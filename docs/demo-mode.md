# デモモード

`/demo` にアクセスすると、サンプルデータ付きのデモユーザーが自動作成されダッシュボードにリダイレクトする。
ログインフォームの入力は不要。

## 環境別動作

### 固定モード (ローカル / CI)

環境変数 `DEMO_EMAIL` が設定されている場合に有効。

| 項目         | 値                                                    |
| ------------ | ----------------------------------------------------- |
| ユーザー     | `DEMO_EMAIL` の値                                     |
| パスワード   | `DEMO_PASSWORD` の値                                  |
| App Password | `DEMO_APP_PASSWORD` の値で固定の App Password を seed |
| 再訪問       | 同じユーザーを再利用 (signIn)                         |
| cleanup      | 対象外 (`@demo.caldav.local` ドメインのみ削除対象)    |

- 毎回同じアカウントでログインできるため、E2E テストで予測可能
- App Password が固定なので CalDAV クライアントの接続テストにも使える

### ランダムモード (preview / 本番)

`DEMO_EMAIL` が未設定の場合に有効。

| 項目         | 値                                       |
| ------------ | ---------------------------------------- |
| ユーザー     | `demo-{ランダム8文字}@demo.caldav.local` |
| パスワード   | ランダム (非公開)                        |
| App Password | seed しない (ダッシュボードから手動生成) |
| 再訪問       | 毎回新規ユーザーを作成                   |
| cleanup      | Cron (毎時) で 24 時間経過後に自動削除   |

## 環境変数

| 変数名              | 必須   | 説明                                            |
| ------------------- | ------ | ----------------------------------------------- |
| `DEMO_EMAIL`        | いいえ | 設定するとデモユーザーのメールアドレスを固定    |
| `DEMO_PASSWORD`     | いいえ | デモユーザーのパスワード (未設定時: `changeme`) |
| `DEMO_APP_PASSWORD` | いいえ | CalDAV 接続用の固定 App Password                |

`DEMO_EMAIL` が未設定の場合、`DEMO_PASSWORD` / `DEMO_APP_PASSWORD` は無視される。

## 環境ごとの設定場所

| 環境                | 設定方法                         | DEMO_EMAIL |
| ------------------- | -------------------------------- | ---------- |
| ローカル            | `.dev.vars`                      | 設定する   |
| CI (GitHub Actions) | GitHub Secrets                   | 設定する   |
| preview デプロイ    | なし (wrangler.jsonc に含めない) | 未設定     |
| 本番                | なし                             | 未設定     |

## サンプルデータ

デモユーザーには以下のカレンダーとアイテムが自動作成される:

| カレンダー   | 種別   | 色      | アイテム数                                                |
| ------------ | ------ | ------- | --------------------------------------------------------- |
| 仕事         | VTODO  | #FF6B6B | 3 (週次レポート作成, クライアントMTG準備, コードレビュー) |
| プライベート | VTODO  | #4ECDC4 | 3 (スーパーで買い物, ジムに行く, 本を読む)                |
| スケジュール | VEVENT | #45B7D1 | 2 (チームスタンドアップ, 1on1ミーティング)                |

## ローカルでの使い方

```sh
# .dev.vars に追加
DEMO_EMAIL=
DEMO_PASSWORD=
DEMO_APP_PASSWORD=
```

```sh
bun run dev
# ブラウザで http://localhost:8787/demo にアクセス
```

CalDAV クライアントから接続:

- サーバー: ${DEMO_EMAIL} (または cloudflared tunnel URL)
- ユーザー名: ${DEMO_PASSWORD}
- パスワード: ${DEMO_APP_PASSWORD} (App Password)

## Cron クリーンアップ

`@demo.caldav.local` ドメインのユーザーを対象に、作成から 24 時間経過したものを削除する。
カレンダー、オブジェクト、変更ログ、App Password、セッションを含む全関連データが削除される。

固定モードのユーザーはドメインが異なるため削除されない。
